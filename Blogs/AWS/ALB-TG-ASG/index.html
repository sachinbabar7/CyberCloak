ALB - The traffic Distribution Engine

Application Load Balancer (ALB) sits at the front, receiving incoming requests and intelligently distributing them across healthy targets. Healthy targets are nothing but pool members where the application is running.

Key configuration elements of ALB include:

  • Listeners: Define ports and protocols (HTTP:80, HTTPS:443)
  • Routing Rules: Content-based routing using host headers, paths, HTTP methods
  • SSL/TLS Termination: Handle certificates and offload encryption from backend instances
  • Sticky Sessions: Route requests from specific users to the same target
  • Cross-Zone Load Balancing: Distribute traffic evenly across all AZs.
  • WAF Integration: Add Web Application Firewall protection.
  • Cognito Integration: Handle user authentication at the load balancer level.
  • Lambda Functions as Targets: Route requests directly to serverless functions.
