Key notes on AWS Cloud Computing Fundamentals

PART A: Multiple AWS Accounts

Ingress Account : Handles incoming traffic from internet

Egress Account : Handles outgoing traffic to internet

Inspection Account : Security inspection of outgoing traffic to internet

Workload Account : Your actual application (compute + database)

Firewall management Account : Controls firewall rules centrally

Certificate account : Manages SSL/TLS certificates

PART B: VPC and Subnets

Public Subnets : Connected to internet

Private Subnets : Not directly connected to internet

Transit Subnets : used to connect to Transit Gateway (TGW)

PART C: Transit Gateway (TGW) - the Core Network Hub

A centeral router connecting all VPCs together

  • Central Control
  • Scalable
  • Cleaner routing

    • PART D: Inspection VPC (Security Layer)

    traffic is inspected before going anywhere

  • AWS network firewall - Blocks malicious traffic, security rules, filters traffic
  • Private + transit subnets

    • PART E: Ingress VPC (Incoming traffic from Internet) = NO DIRECT ACCESS TO WORKLOAD VPC

    Flow inside ingress

    Internet ==> AWS WAF (filters attacks) ==> AWS Network Firewall ==> Application Load Balancer (ALB) ==> Transit Gateway (TGW) ==> Workload VPC

    Components

  • WAF -> block web attacks
  • Shield Advanced - DDoS protection
  • ALB => Distribute traffic to backend

    • PART F: Egress VPC (Outgoing traffic to Internet)

    APIs/Updates/External services

    Flow inside egress

    Workload VPC ==> Transit Gateway (TGW) ==> Egress VPC ==> AWS Network Firewall ==> NAT Gateway ==> Internet

    PART G: Workload VPC (Application traffic)

    Components

  • EC2 instances (containers)
  • ALB ==> routable subnet
  • Database ==> Fully isolated/ no direct access to internet / non-routable subnet

    • PART H: Certificate & TLS inspection

    Certificate management

    Certificate account ==> ACM (AWS Certificate Manager) ==> TLS inspection in AWS Network Firewall

    End to End Traffic flow

    CASE 1: Internet to Application

    Internet ==> Ingress VPC ==> WAF ==> Network Firewall ==> ALB ==> TGW ==> Workload VPC ==> App (compute) ==> Database

    CASE 2: Application to external API

    Workload VPC ==> Transit Gateway (TGW) ==> Egress VPC ==> AWS Network Firewall ==> NAT Gateway ==> Internet

    Design considerations

  • Zero Trust Network Access (ZTNA)
  • Centralize Security in Inspection VPC
  • Least Privilege Access => Only required connections are allowed
  • Separation of Duties => Network, Security & Apps teams have separate accounts
