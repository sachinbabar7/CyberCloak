PART B: What is DNSSEC?

  • DNSSEC is a set of protocol extensions that protect DNS by enabling resolvers to verify that DNS responses are authentic and have not been altered.
  • DNSSEC mitigates this risk by allowing DNS data to be digitally signed and cryptographically validated.
  • DNSSEC also provides origin authentication, ensuring that the DNS data comes from the legitimate source.
  • DNSSEC is about trust, verifying that the DNS answer you received is the one the domain owner intended to publish.

    • DNS Cache Poisoning

    DNS cache poisoning is a type of attack that exploits vulnerabilities in the Domain Name System (DNS) to redirect users to malicious websites or servers. In this attack, an attacker sends forged DNS responses to a DNS resolver, which then caches the incorrect information. As a result, when users attempt to access a legitimate website, they are instead directed to a malicious site controlled by the attacker.
