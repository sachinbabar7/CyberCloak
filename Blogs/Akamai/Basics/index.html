PART B: What is DNSSEC & RRSIG Records?

  • DNSSEC is a set of protocol extensions that protect DNS by enabling resolvers to verify that DNS responses are authentic and have not been altered.
  • DNSSEC mitigates this risk by allowing DNS data to be digitally signed and cryptographically validated.
  • DNSSEC also provides origin authentication, ensuring that the DNS data comes from the legitimate source.
  • DNSSEC is about trust, verifying that the DNS answer you received is the one the domain owner intended to publish.
  • Benefits of DNSSEC:
    • Prevents DNS Cache Poisoning
    • Ensures Data Integrity
    • Provides Origin Authentication
    • Supports Secure Delegation
    • Enhances Overall DNS Security
    • Authenticity: Did this DNS data really come from the domain owner?
    • Integrity: Was the data altered while traveling across the internet?

DNS Cache Poisoning

DNS cache poisoning is a type of attack that exploits vulnerabilities in the Domain Name System (DNS) to redirect users to malicious websites or servers. In this attack, an attacker sends forged DNS responses to a DNS resolver, which then caches the incorrect information. As a result, when users attempt to access a legitimate website, they are instead directed to a malicious site controlled by the attacker.

DNS cache poisoning can have serious consequences, including:

  • Data and Identity Theft: harvest login credentials and personal information
  • Malware Distribution: redirect users to sites that host malware
  • Phishing Attacks: redirect users to fake websites that mimic legitimate ones
  • Denial of Service (DoS) Attacks: redirect users to non-existent or malicious servers, causing service disruptions
  • Reputation Damage: damage the reputation of legitimate websites and organizations

How DNSSEC Works

DNSSEC strengthens DNS by adding digital signatures based on public key cryptography. Instead of blindly trusting responses, resolvers can verify them.

  • Each DNS zone has a private key and a public key
  • The private key is used to sign DNS records.
  • The public key is published in DNS and used for verification.

When a DNSSEC-enabled resolver receives a response:

  • It retrieves the record’s digital signature (RRSIG)
  • It retrieves the zone’s public key (DNSKEY)
  • It validates the signature against the data
    • Valid signature → response is trusted
    • Invalid or missing signature → response is rejected

    Importantly, DNSSEC signs the data itself– not individual queries– therefore, caching and performance remain intact.
