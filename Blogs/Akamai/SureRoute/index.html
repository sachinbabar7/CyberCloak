Akamai SureRoute

SureRoute is a feature provided by Akamai that helps optimize the delivery of content by selecting the best route for data to travel from the origin server to the end user. It is designed to improve performance and reliability by dynamically choosing the most efficient path for data transmission.

SureRoute works by continuously monitoring the network conditions and performance metrics of various routes between the origin server and the end user. It uses this information to make real-time decisions about which route to use for delivering content. The goal is to minimize latency, reduce packet loss, and ensure a smooth user experience.

By leveraging SureRoute, Akamai can help ensure that content is delivered quickly and reliably, even in the face of network congestion or other issues that may arise along the way. This can be particularly beneficial for websites and applications that require high performance and low latency, such as streaming media, online gaming, and e-commerce platforms.

Observations: Application hosted (abc.com) behavior behind multi DC with GTMs/LTMs having different origin servers and edge servers load the traffic in zig-zag patter or Oscillation pattern.

This behavior shows the one DC will have increase in traffic load compared to other DCs & when other DC pickups up, the load is unbalanced.

Traffic monitoring traffic shows the following:

  • SureRoute is Core feature of Akamai CDN.
  • It test multiple routes between Akamai edge servers and origin servers to identify the best path.
  • It helps with Enhanced Akamai Protocol:
  • Increase the size of congestion window
  • Use of selective acknowledgments (SACKs) to improve performance over high-latency connections
  • Uses "duplicate ACK". This is dual acknowledgment that's sent by the receiver if a data packet is lost. It forces quicker retransmission of lost packets.
