Physical to Virtual migration steps

PUBLISHED ON NOV 11, 2025 / 13 MIN READ
  • LTM - Physical to Virtual Migration
    • Take UCS backup from old box
    • Master key backup
    • Configure MGMT on new box
    • Configure licennse on new box
    • Configure network (vlan/trunk) on new box
    • Configure system provisioning on new box
    • Restore Master-Key on new box
    • Restore UCS on new box

Introduction

Hardware to virtual edition - Through put 200mbps.

The summary of entire process is:

OPEN IMAGE IN A NEW TAB AND IT WILL BE THE HIGH RESOLUTION IMAGE


>Take UCS backup from old box

  • Backup the UCS file:

    • [root@bigip1:Active:Standalonoe] confiig # tmsh save sys ucs $(echo $HOSTNAME | cut -d'.' -f1)-$(date +%H%M-%m%d%y)

  • UCS path:

    • /var/local/ucs

  • SCP file from F5 to any jump server to retreive using WinSCP or FileZilla

    • scp admin@hostname:/var/local/ucs/HOSTNAME-045819NOV2025.ucs /home/

    OR

    scp -O admin@hostname:/var/local/ucs/HOSTNAME-045819NOV2025.ucs /home/

  • Copy file from local JUMP host server to local machine using WINSCP

>Master key unit (MKU) backup

  • Old F5, Copy the key

    • [root@bigip1:Active:Standalonoe] confiig # f5mku -K

    tpWnKcudbfdfdlerTFSDSFpkFSDF

  • New F5, paste the key

    • [root@bigip1:Active:Standalonoe] confiig # f5mku -r tpWnKcudbfdfdlerTFSDSFpkFSDF

  • Verify master key is same on both system.

>Restore UCS file

  • New F5 box, load the UCS. UCS file contain license also, since we loaded license in initial stage, we enter "no-license" while restoring UCS; it simply ignore the license file from UCS

    • [root@bigip1:Active:Standalonoe] confiig # tmsh load sys ucs /var/tmp/HOSTNAME-045819NOV2025.ucs no-license platform-migrate

  • Verify the UCS

    • [root@bigip1:Active:Standalonoe] confiig # load sys config verify

  • Save config.

    • [root@bigip1:Active:Standalonoe] confiig # save sys config
